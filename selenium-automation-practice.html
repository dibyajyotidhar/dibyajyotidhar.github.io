1. Project Overview

This is a practice automation project developed using Selenium WebDriver with Java. The objective of this project was to understand UI automation workflows, element handling, synchronization, and basic validation using assertions.

2. Test Scenarios Covered
  • Login functionality (positive and negative cases)
  • Form input handling and submission
  • Page navigation validation
  • Input field validation
  • Button interactions and UI behavior
3. Current Structure (Learning Phase)
  • Java-based Selenium test scripts
  • Direct WebDriver usage
  • No framework pattern implemented yet
  • Basic assertions for validation
4. Sample Code Implementation

Example Selenium login automation snippet: 

                
                // Login test example
                driver.findElement(By.id("username"))
                    .sendKeys("testuser");

                driver.findElement(By.id("password"))
                    .sendKeys("password");

                driver.findElement(By.id("login"))
                    .click();

                // Basic validation
                Assert.assertTrue(
                    driver.getCurrentUrl()
                        .contains("dashboard")
                );

Full implementation is available in the GitHub repository.

5. Limitations
  • Page Object Model (POM) not implemented
  • Assertion coverage is limited in some scenarios
  • No reporting integration (Extent / Allure)
